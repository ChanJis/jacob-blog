Nginx 静态资源服务器配置

  1. 创建静态 html 文件
  • 假设nginx 安装目录为：usr/local/nginx， nginx 目录下创建目录

编辑 index.html 文件

  1. 修改 config/nginx.conf

说明：

  1. server 配置
  • listen: 监听端口
  • server_name: 服务名称，对于 listen 端口相同的多个 server, 通过获取 http request 中的 host 来与 server_name做对比，决定使用哪个server
    • 匹配规则（按优先级排序）：
      • 精确名称
      • 星号开头的最长的通配符名称，例如*.example.com
      • 星号结尾的最长通配符名称，例如mail.*
      • 第一个匹配正则表达式（按配置文件顺序）
      • 使用 default server
      • 第一个匹配到listen 端口的第一个server(按配置文件顺序)
  • location: 根据用户请求的网站URL进行匹配，匹配成功即进行相关的操作
    • 语法： location [ = | ~ | ~* | ^~] uri {...}
      • none: 最前缀匹配, 如果匹配到，还会往下（配置顺序）搜索
      • =： 精确匹配，如果匹配到，则会停止搜索
      • ~： 大小写敏感的正则匹配，如果匹配到，则会停止搜索
      • ~*： 大小写不敏感的正则匹配，如果匹配到，则会停止搜索
      • ^~： 非正则匹配，正则匹配将失效，如果匹配到，则会停止搜索
    • location 块（静态文件请求）
      • alias 和 root 的区别
        • alias:
          1. 必须以 ‘/‘结尾
          2. 处理结果：root路径+location的路径
        • root:
          1. 结尾’/‘可加可不加
          2. 处理结果：使用alias路径替换 location 路径
  1. 压缩配置
  • gzip: 开启 gzip 压缩
  • gzip_min_length: 对于小于 gzip_min_length 字节的数据不进行压缩
  • gzip_comp_level: 压缩类型，数值在（1-9）之间，9表示最慢但是压缩比最大，1表示最快但是压缩比最小
  • gzip_type: 需要被压缩数据的格式

server_name 正则表达式写法：

  • 以 ~^ 开头，以 $结尾
